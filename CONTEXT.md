- 항상 수정 이유와 변경된 부분을 명확히 설명하라.  

## 1. 개발 원칙
- **MVP 최적화**: 재현성(reproducibility) 우선
- **코드 제안 단위**: 함수/메서드/클래스 단위
- **코드 위치**: 셀 번호 기준으로 제안
- **로깅**: `logger.debug` 최소화, 로깅 우선
- **코드 길이 규칙**
  - MVP
  - 10라인 이하 → 바로 제안 가능
  - 10라인 이상 → 반드시 확인 질문 후 제안
- **스타일 가이드**
  - PEP 8 + Black + isort
  - PEP 484 (타입힌트), PEP 257 (Docstring)
- **pip 헬퍼 모듈 활용**: `helper-dev-utils`, `helper-plot-hangul` 존재 시 적극 활용
- **파이선**: 반드시 conda mis18 사용 (python 3.11)

## 2. 상호작용 프로토콜

5. 단문·20라인 이하 → 확인 질문 생략 가능
