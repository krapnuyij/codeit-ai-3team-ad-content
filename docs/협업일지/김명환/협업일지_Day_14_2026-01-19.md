## [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?

**답변**:  

```text
1. LLM 기반 HTML 텍스트 생성의 장점
   - AI가 광고 컨텍스트를 이해하고 적절한 레이아웃 제안 가능
   - HTML/CSS로 복잡한 스타일링 및 애니메이션 효과 구현 가능
   - 한글 폰트 처리 및 텍스트 배치 품질 대폭 향상
   - 디자인 일관성 및 자동화 수준 향상

2. HTML to PNG 렌더링의 기술적 과제
   - 브라우저 렌더링 엔진 통합 (Playwright/Selenium 등)
   - 렌더링 성능 및 이미지 품질 최적화 필요
   - 폰트 로딩 및 CSS 적용 타이밍 이슈
   - 해상도 및 DPI 설정 최적화

3. FLUX 모델 실험 결과
   - FLUX.1 다양한 설정으로 테스트했으나 일관성 부족
   - FLUX.2가 품질 면에서 우수하나 모델 사이즈 문제 존재
   - Qwen-FLUX 조합 실험으로 텍스트 이해도 향상 시도
   - 각 모델별 장단점 및 적용 시나리오 파악

4. 통합 파이프라인 구축 진전
   - core/engine.py, processors.py, worker.py 통합 완료
   - models/llm_text.py 신규 모듈로 텍스트 생성 분리
   - Docker 환경 업데이트로 배포 준비 완료
   - 전체 워크플로우가 안정화 단계 진입

5. 발견한 문제점
   - LLM 프롬프트가 단순하여 광고 느낌이 약함
   - 초기 이미지 생성 엔진의 품질 편차 존재
   - 렌더링 속도와 품질의 트레이드오프 조정 필요
```
