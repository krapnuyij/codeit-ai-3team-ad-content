**답변**:  
```
1. FLUX.1 vs FLUX.2 품질 비교
   - FLUX.1 (Kontext): 이미지 합성 품질이 요구사항 미달
     - 제품 원형 변형 또는 배경 합성 부자연스러움 발견
     - 단일 모델 접근의 한계 확인
   
   - FLUX.2: 이미지 합성 품질이 요구사항 충족
     - Hugging Face 테스트 페이지 결과 매우 긍정적
     - 제품 원형 유지 및 자연스러운 배경 생성 확인
     - 실제 광고 이미지 생성에 적합한 수준

2. 모델 사이즈 vs 품질 트레이드오프
   - FLUX.2 모델 사이즈가 매우 크다는 문제점 발견
   - 양자화 없이는 실제 서비스 배포 어려움
   - 4bit 양자화가 필수적인 최적화 방안으로 확인
   - 양자화 후 품질 유지 여부가 핵심 검증 포인트

3. HTML 기반 텍스트 렌더링의 가능성
   - AI LLM으로 HTML/CSS 생성 → 렌더링하는 방식 검토 완료
   - 장점:
     - 한글 폰트 및 스타일링 제어 용이
     - 텍스트 배치 및 레이아웃 자동화 가능
     - LLM이 디자인 컨텍스트를 이해하고 적절한 HTML 생성
   - 고려사항:
     - HTML 렌더링 성능 및 이미지 변환 품질
     - 브라우저 엔진 또는 렌더링 라이브러리 선택

4. 기술 스택 진화 방향
   - 이미지 생성: FLUX.1 → FLUX.2 (품질 우선)
   - 메모리 최적화: 4bit 양자화 필수
   - 텍스트 렌더링: PIL/Pillow → HTML/CSS + LLM
   - 전체적으로 품질 향상을 위한 기술 업그레이드 진행 중
```
