**답변**:  
- 템플릿 상속 구조를 활용하여 헤더/푸터 중복 코드를 제거하고 유지보수성을 높이자는 제안을 하여 팀원들의 동의를 얻음.
- 향후 API 연동 시 JSON Response와 HTML Response를 명확히 분리하여 설계할 것을 제안함.

## [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?
**답변**:  
- 인사이트: FastAPI의 의존성 주입(Dependency Injection) 기능을 활용하면 OpenAI 클라이언트나 데이터베이스 세션을 효율적으로 관리할 수 있음을 확인.
- 문제점: 현재는 Mock 데이터를 사용하고 있어 실제 사용자 데이터 흐름을 테스트할 수 없음. 데이터베이스 연동이 시급함.
