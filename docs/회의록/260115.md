## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 아키텍처 및 파이프라인 설계, 이미지 생성 모델 추가 실험
- 김민혁: 텍스트 생성 모델 작업
- 박지윤: 중간 발표 준비, 통합 테스트
- 이건희: 프론트엔드/백엔드/모델서버 연결, 이미지 생성 모델 추가 실험
- 이솔형: 이미지 생성 모델 작업

### 2. 핵심 논의사항
**주요 이슈**:
- Canny Edge Detecion을 사용하여 누끼따서, 이를 베이스로 3D 렌더링 등 효과 주는 방식 제안
  - 해외 AI 엔진들이 영문 기반이라, 이미지와 한글 텍스트를 합성 또는 동시 생성 시 품질 저하 발생으로 인한 해결책
  - 마스킹으로 배경을 제거하고 AI가 해당 영역을 변형하도록 유도하고 있으나,
  - 검은색 배경 처리가 깔끔하지 않고 물체 위치 잡는 데 어려움 있음
  - 창문 옆, 탁자 옆 등 사물의 위치를 인식해 텍스트를 자동배치하는 엔진 검토 중, 한글 호환성 문제로 난항
- 단일 모델 제안: HighCWu/FLUX.1-Kontext-dev-bnb-hqq-4bit
  - 기존에는 배경 생성과 오브젝트 합성을 분리했으나, 단일 보델(Image-to-image)로 처리하는 방안 제안
  - 빵 이미지를 입력했을 때, 원본의 특성을 유지하면서 그림자나 배경이 자연스럽게 생성됨 확인
  - 파이프라인이 단순해지고, 4bit 양자화 모델 사용 시 VRAM 약 12GB로 효율이 좋음
  - 제품(오브젝트) 원형 변형 우려 제기하였으나, 예시를 통해 원본 유지 가능성을 입증
  - 긍정적으로 검토 예정
- 프론트엔드-백엔드-모델서버 연결 성공
  - 홈페이지 생성 기능 연동 확인 완료
- 택스트를 단순히 지우는 것 뿐만 아니라, 자연스럽게 합치는 기능 진행 중
  - DeepLabV3를 사용하여 이미지 내 빈 공간(가장 점수가 높은 영역)을 찾아 텍스트 배치 좌표 산출
  - SDXL 사용하여 한글 텍스트를 이미지 분위기에 맞게 스타일링하여 삽입하는 방식 진행 중
  - 이미지 생성 시 한글을 동시에 입력하는 방식은 글자 깨짐 현상이 심하여,
  - 현재처럼 생성 후 합성 방식을 유지하되 품질을 높이기로 함
- 이미지 생성 시 Negative Prompt를 보강하여 원치 않는 텍스트가 생성되는 것을 막도록 함
- 이미지 생성 시 FLUX 모델 실험 진행 중
- 이미지 생성 모델 코드 GitHub에 업로드
  - alpha 브랜치에 Merge 예정
- requirements.txt에 존재하지 않는 버전이 적힌 패키지 있음, 수정 예정
