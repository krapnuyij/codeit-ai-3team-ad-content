## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 파이프라인 설계
- 김민혁: 텍스트 생성 모델링
- 박지윤: 전체 흐름 파악 및 QA, 일정 및 진행 사항 관리
- 이건희: 백엔드 구축
- 이솔형: 이미지 생성 및 합성 파이프라인 정비

### 2. 핵심 논의사항
**주요 이슈**:
- MCP는 LLM과 바로 연결되는 구조가 아님
  - 필요한 구성은 MCP 서버, MCP 어댑터, MCP 어댑터와 연결된 LLM이지만, Claude 외 다른 LLM은 별도 어댑터 개발 필요
  - 실제 적용 가능 여부는 추가 검토
- 서빙 서버 기능 인터페이스는 구현 완료
  - Step 1에서는 시간 체크 진행 가능 설계, Step2, Step3 남은 시간 계산 부정확
  - 시간 계산 로직은 2~3주 차에 보완 예정
- Flux 기반 생성/합성 테스트
  - 이미지 생성: Flux 엔진 이미지 생성 품질은 매우 우수, Inference step(생성 카운트) 수 3->28회 증가 후 사실성 향상
  - 이미지 합성: 일관성 붕괴. 이미지가 합성되는 것이 아닌 프롬프트만 다시 적용된 결과처럼 보임
  - 이미지 합성 실패 원인: Flux 파이프라인 바이브 코딩의 한계. 공식적으로는 파이프라인 직접 연결을 지원하지 않음
- 메모리/양자화
  - L4 환경에서 8bit 양자화 시 OOM 발생
  - 4bit 양자화 시 파이프라인 연결 불안정
  - 동시에 올리면 4bit 필요, 분리 운영 시 8bit 가능
- Qwen(Qwen-VL) 기반 위치 자동 판단 테스트
  - 이미지 분위기·구성 이해도가 매우 높음, 텍스트 위치를 적절한 배치 영역으로 자동으로 제안 가능
  - 텍스트를 어디에 놓을지를 프롬프트 엔지니어링이 아니라 LLM 판단으로 해결
  - 테스트 단계로 아직 파이프라인에는 미통합
  - 자동 배치 로직 후보로 추가 실험 필요
- 텍스트 생성 모델
  - 다양한 경우 테스트 시행
  - 타이틀 영역 박스·마스크 자동 추출, 일부 이미지에서 마스크 범위 오류 수정 중
  - FIN 모델 사용 시 OOM 발생, SDXL 계열로 다운그레이드하여 안정화 진행
  - 정상 동작 확인 후 파이프라인에 기능 병합 예정 및 텍스트를 폰트가 아닌 이미지 생성 방식으로 확장 검토 예정
- 이미지 생성 기능을 백엔드에 추가 작업 진행 중
