## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 파이프라인 및 아키텍처 구축, 서빙 서버 구축
- 김민혁: 텍스트 생성 모델링
- 박지윤: 테스트, 발표 자료 제작, 진행 사항 및 일정 체크
- 이건희: 불참
- 이솔형: 이미지 생성 모델링

### 2. 핵심 논의사항
**주요 이슈**:
- 홈페이지 제너레이터: 이미지 생성모델과 연동 완료
  - 전체 생성 시 1시간 이상 소요
  - 자동 생성되는 틀을 보여주는 데 의의를 둠
- 채팅 기반 광고 생성기
  - 기존 광고에 추가 텍스트를 입력하여 재생성하는 기능 구현 성공
- 대시보드에 모델(SDXL/FLUX)을 선택하여 테스트할 수 있는 옵션 추가
- 프롬프트 최적화
  - 프롬프트에 중요 규칙, 100단어 이상 묘사 등의 지시어를 넣었더니 품질이 급격히 좋아짐
  - 한글로 쓰라고 명시하지 않으면 영어가 나오거나, 토큰 제한(77토큰 등)으로 인해 뒷부분이 잘리는 현상이 있었으나 해결함
- Qwen 모델 활용
  - 기존 Segment Anything 모델 등으로 텍스트 위치를 직접 찾으려고 했으나 복잡함
  - Qwen모델이 이미지의 공간, 색감, 분위기를 분석해 줌
  - 로직: 사용자가 자동으로 위치를 설정하면 Qwen이 분석한 정보를 바탕으로 LLM이 텍스트 위치를 결정, 수동으로 설정하면 그에 따름
  - process.py 코드 참고하여 텍스트 생성 로직에 통합 예정
- 이미지 모델 테스트 결과
  - SDXL: 프롬프트를 아주 디테일하게 줘도 엉뚱한 결과(위치 못 잡음, 이미지 왜곡)가 많이 나옴
  - FLUX v2: 배경 품질은 매우 좋으나, 위치 잡기나 세밀한 제어가 어려움. 하지만 광고용으로 쓸만함
  - FLUX v1(확정): 가장 안정적임
  - 건어물로 테스트를 하였고, 사과로 하면 품질이 훨씬 좋을 것으로 예상, 사과로 다시 테스트 예정

**논의된 해결방안**:
- SDXL vs FLUX
  - SDXL은 동일한 프롬프트에서 원하는 이미지가 잘 나오지 않음 (예: 사과를 그렸는데 캔디처럼 나옴)
  - FLUX 모델이 생성시간이 길긴 하지만(약 2~3분), 훨씬 안정적이고 고품질
  - FLUX v1모델을 메인 모델로 확정
  - SDXL은 옵션으로 남겨두기로 함

**팀 결정사항**:
- 아키텍처 최종 수정 예정
  - 최종 파이프라인 구성: FLUX(배경) + Qwen(이미지 분석) + LLM(텍스트 렌더링)
- 정량적 평가: CLIP Score
  - API에 CLIP Score 측정 기능 추가 예정
  - 생성된 이미지와 프롬프트 일치도를 점수화
- 정성적 평가: 시각적 매력도, 상품 식별성, 상업적 활용도, 메시지 일관성 등의 항목(Rubric)
  - 1~5점 척도로 직접 점수를 매겨 평균을 냄
  - 설문지 만들고 팀원들이 투표하여 점수를 낼 예정
- 모델링 테스트 과정에서 비교 이미지와 최종 이미지 전달 예정
- 시연 영상 예정
  - 이미지 모델링 프롬프트 공유 요청
