## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 프롬프트 엔지니어링, 아키텍처 및 파이프라인 설계
- 김민혁: 텍스트 생성 모델
- 박지윤: 테스트, 진행 사항 및 일정 체크
- 이건희: 불참
- 이솔형: 불참

### 2. 핵심 논의사항
**주요 이슈**:
- HTML 렌더링 방식 테스트
  - GPT-5가 GPT-5mini보다 화려하고 효과적인 CSS 코드 생성
  - 기존 AI 모델 생성 방식보다 글자가 훨씬 선명하고, 그라데이션이나 3D 효과 같은 디자인 요소 적용 가능
  - 현재 테스트 결과물은 배경이 다소 어둡고 텍스트 효과가 단순하여 개선 필요
  - 단순 텍스트 나열이 아니라, 특가 세일/리본 효과/폭죽 효과 등 광고 전단지 느낌이 나도록 프롬프트 연구 필요
  - 현재 기본 포트로 출력되는 문제를 해결하기 위해, 서버 내부의 폰트 파일 목록을 조회하여 LLM에게 전달하고,
  - LLM이 분위기에 맞는 폰트를 선택하도록 로직 개선 제안
- FLUX v2 양자화 시도하였느나 성능 확보가 어려워 FLUX v1으로 진행 중
- 텍스트 모델링 3가지 방식 비교 테스트:
  - 1. GPT가 직접 이미지 생성
    2. HTML/CSS 렌더링 합성
    3. 기존 AI 모델 합성
  - HTML/CSS 방식과 모델 합성 방식 모두 나쁘지 않으나, 정보 전달(가격, 주소 등) 측면에서 HTML 방식이 유리함
- 이번주 금요일까지 모든 기능 개발 및 결과물 생성 완료 목표
  - 금요일에 시연 동영상 녹화 예정
- 아키텍처 문서 수정 예정
  - 텍스트 생성 파트를 SDXL에서 LLM HTML 렌더링으로 수정하여 업데이트 예정

**논의된 해결방안**:
- HTML 렌더링 방식 채택
  - process.py 내 use_llm_text 옵션을 True로 설정하여 LLM 기반 HTML 렌더링 방식을 메인으로 사용하기로 결정
  - 복잡한 텍스트 효과(제목)와 정보성 텍스트(주소, 가격 등) 모두 HTML/CSS로 처리
  - 배경 이미지 위에 텍스트가 잘 보이도록 반투명 배경 박스나 하단 정보란을 HTML 레이아웃으로 구성
  - AI 모델을 이용한 텍스트 질감 처리는 시간 관계상 2순위로 미룸
 - 타겟 상품을 과일로 변경
   - 건어물을 학습 데이터 부족으로 AI가 생성한 이미지 품질이 낮음
   - 과일(사과, 바나나 등)은 학습 데이터가 풍부하여 고품질 이미지가 잘 나옴
   - 남은 기간 동안 퀄리티를 보장하기 위해 명절 맞이 사과 특가 세일 광고를 메인 시나리오로 정함
   - 광고 콘텐츠를 명절 대목을 맞은 청과물 시장의 사과 세일 전단지
   - 명절 세일/특가 등 메인 문구와 할인 가격, 원산지, 점포명, 주소, 전화번호 등 정보 포함

**팀 결정사항**:
- 평가 지표 중 하나로 광고 품질 검사 기능 시 LLM에게 입력된 프롬프트와 결과물을 주고 점수를 매기게 하는 방식
