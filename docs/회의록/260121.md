## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 파이프라인 구축, 서빙 서버 구축
- 김민혁: 텍스트 생성 모델
- 박지윤: 진행사항 및 일정 체크, 발표 자료
- 이건희: 프론트엔드, 백엔드 구축
- 이솔형: 이미지 생성 모델

### 2. 핵심 논의사항
**주요 이슈**:
- SDXL 엔진 선택적으로 할 수 있게끔 추가
- 이미지 생성 단계 로직 개선
  - 배경만 생성, 전체 생성, 텍스트만 입히기 등 선택적으로 실행할 수 있도록 로직을 수정함
  - 배경 생성 단계까지 테스트 완료
- Docker환경에서 백엔드와 AI 서버를 연동하여 점포 홈페이지 생성 기능까지 연결 성공
- Docker 컨테이너를 재시작하면 생성했던 데이터베이스가 초기화되는 문제 발생
  - Docker 설정을 수정하여, 컨테이너를 내렸다 올려도 데이터가 유지되도록 조치함
- 생성된 홈페이지로 이동하는 링크가 단순 텍스트로 표출되는 버그 수정함
- 홈페이지 생성기(Homepage generator) 구현
  - MCP tool처럼 LLM에 도구를 연결하여, 사용자가 요청하면 템플릿 기반으로 홈페이지를 생성하고 저장/로딩하는 기능
  - 페이지 생성은 되지만, 이미지를 업로드하거나 텍스트를 입력하는 UI가 부족함
- 채팅 기반 광고 생성기 추가
  - 현재 이미지 기반 생성이 불안정하여, 채팅(텍스트 프롬프트) 기반으로 광고를 생성하는 기능을 임시로 추가함
- Qwen 엔진 도입
  - 이미지의 공간 분석과 색삼/재질 분석을 수행하여, 이를 텍스트 생성 시 LLM 프롬프트에 반영하도록 함
  - 예: 사과가 왼쪽에 있으니 텍스트는 오른쪽에 배치
- SDXL_Generator와 별도록 Flux_Generator를 코드에 추가함
- HTML 기반 텍스트 생성 후 파이프라인에 반영하는 작업 중
- FLUX.2 kein 모델로 이미지 생성 시도
