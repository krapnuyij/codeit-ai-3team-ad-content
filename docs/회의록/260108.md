## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: MCP-LLM-서빙 자동화 진행
- 김민혁: 텍스트 처리 모델링
- 박지윤: 전체 흐름 파악, 테스트 진행
- 이건희: 프론트앤드, 백엔드 구축 진행
- 이솔형: 이미지 생성 모델링

### 2. 핵심 논의사항
**주요 이슈**:
- MCP 서버 + GPT-5 LLM 연결용 어댑터 직접 구현
  - 구조는 RestAPI까지는 그대로
  - LLM이 MCP를 통해 서빙 서버 호출, 상태 업데이트, 완료시 광고 이미지 저장까지 지시 가능까지 실제 동작 확인
  - 자동화 가능성 확인하여 소스를 알파 브랜치에 업로드
  - 파라미터 튜닝 필요, 한글 깨짐 문제, 바이브 코딩 수준으로 구조 정리는 미완
  - LLM 테스트 시 서빙 서버만 실행,, MCP 서버는 어댑터에서 자동 실행
  - 백엔드 + 서빙 서버를 동시에 띄우는 구조는 정리 중
- 텍스트 처리 시 LLaMA 계열 AI를 이용한 정제 방식
  - 한글, 영어 텍스트 여러 경우 기존 SDXL 모델 테스트 결과 글씨 제거가 부자연스러움
  - 1차로 LLaMA 계열 AI를 이용하여 누끼와 텍스트 제거, 2차로 SDXL로 자연스럽게 리터치 방식 진행
  - Qwen 사용 고려
- 이미지 모델링 개선/미세 조정 진행
  - 배경 합성 시 그림자 처리, 톤 매칭, 색감 조정 진행
  - 오브젝트 테두리 경계 부자연스러움 개선 진행, 외곽선 블렌딩 튜닝
- 사용자 입력 관점 정리
  - 단일 대상 광고에서 확장 고려, 매장 정보 기반 입력 구조 고려
  - 이미지 생성 결과를 백엔드에서 받는 방법 파악
- alpha 브랜치 테스트 기준 부재
  - 도커로 묶은 후 통합 테스트 진행 가능
