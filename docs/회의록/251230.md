## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 아키텍처 설계 및 파이프라인 구성
- 김민혁: 상품 이미지를 다른 광고 홍보물 레퍼런스에 적용시키는 모델 작업중.
- 박지윤: 맥에서 GCP VM을 vsvode에 연결하는데, 맥 환경이라 어려움을 겪는 중.
- 이건희: 페이지 제작으로 ui구성 및 사용자 구별 작업
- 이슬형: 이미지 생성 및 조합 모델 개발

### 2. 핵심 논의사항
**주요 이슈**:
- 팀 협업을 위한 팀회의록 작성 지시 및 GitHub 운영 방식 정리 필요
- 서비스 운영 환경에서 메모리 최적화(양자화, 스왑 메모리 설정)가 필요함
- 이미지 제작 파이프라인에서 마지막 한글 텍스트 삽입 시 품질(깨짐/가독성) 이슈
- 광고 이미지 생성 과정: 누끼(배경 제거) → 배경 생성(양자화 모델) → 위치 지정 → 텍스트 기반 합성 → 텍스트 추가의 전체 흐름 확정 필요
- 모바일 사용자 대응을 위한 모바일용 페이지(UI/UX) 구현과 세션 관리 방식(JWT 적용) 논의
- 여러 사용자를 위한 서비스 제공을 위해 로그인 기능(권한/계정 관리) 구현 필요

**논의된 해결방안**:
- GitHub 운영: 브랜치 전략/PR 규칙/이슈 템플릿을 정하고, 회의록은 고정 포맷으로 남겨 관리
- 메모리 최적화:
    - 모델 양자화(예: 4bit/8bit) 적용 후보 검토
    - GCP VM에서 스왑 메모리 생성 및 튜닝으로 OOM 방지
    - “고품질(무거운) 모델은 서버에서, 경량 처리는 클라이언트/비동기로” 역할 분리 검토
    - 한글 텍스트 품질: 생성 모델로 직접 쓰기보다 마지막 단계에서 폰트 렌더링 방식으로 합성해 깨짐 최소화
- 이미지 파이프라인:
    - 누끼 모델/도구 선정 → 배경 생성 모델 선정(양자화 적용 가능)
    - 위치/레이아웃 규칙을 템플릿화하고, 텍스트 입력에 따라 자동 합성되도록 설계
- 모바일 페이지/세션:
    - 모바일 UI 우선 설계(반응형 또는 모바일 전용)
    - JWT 기반 인증/세션 관리 도입해 로그인 유지 및 API 보호
- 로그인 기능:
    - 사용자 계정/권한/데이터 분리(개인별 생성물 저장) 기본 구조 먼저 구현

**팀 결정사항**:
- 회의록 포맷 통일 및 GitHub 협업 규칙(브랜치/PR/이슈) 정리 후 적용
- 초기 MVP는 고품질 우선으로 “무거운 최신 모델” 중심으로 설계하되, 운영 안정화를 위해 양자화 + 스왑 메모리를 함께 적용하는 방향으로 진행
- 광고 이미지 생성은 누끼 → 배경 생성 → 배치/합성 → 마지막 한글 폰트 렌더링 삽입의 파이프라인으로 확정
