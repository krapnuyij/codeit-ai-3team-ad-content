## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: 아키텍처 점검, 서빙 서버 관리
- 김민혁: 텍스트 처리 및 텍스트 생성
- 박지윤: 전체 흐름 관리, 일정 및 진행 상황 관리
- 이건희: 프론트엔드, 백엔드 구축
- 이솔형: 이미지 생성

### 2. 핵심 논의사항
**주요 이슈**:
- LLM 가이드 문서 작성
  - AI 모델에 프롬프트를 보낼 때 단순히 글자만 넣는 게 아니라, 특정 변수나 문법을 지켜야 한다는 가이드 배포
- GPT 모델 교체: GPT-5mini
- 메모리 이슈: 메모리가 쌓여서 생긴 문제를 해결해기 위해 고도화
- 아키텍처 점검 예정 및 시퀀스 다이어그램 검토 예정
- VM의 GPU 메모리가 24GB보다 작은 22~23GB 정도로 이미지와 텍스트 합성 단계에서 21GB까지 올라서 OOM 위험
- 텍스트 제거 및 재삽입 구현 논의
  - 이미지 생성 단계에서 기존 포스터를 이용하는 등의 이유로 기존 불필요한 텍스트 제거 후 광고 문구를 다시 넣는 기능 구현
  - PIL 라이브러리를 이용하여 기존 글자를 지우고 그 위치에 광고 문구를 올리는 방식
  - AI가 추천하는 방식으로 이미지를 분석하여 어울리는 색상/폰트를 정하도록 도입 예정
  - 기본 흐름(생성 후 합성)을 먼저 구현하고, 효과나 옵션은 추후 보강하는 방식 제안
  - 이미지 생성 시 nagetive 프롬프트에 텍스트 제거 넣어서 텍스트 없이 이미지 생성 제안
- 제품 이미지와 배경 합성 시 Flux kontext 라는 모델 실험하였으나 용량이 커서 실패
  - 양자화 모델 찾는 중
- 홈페이지 생성 시연
  - DB 연동 완료 (고객 ID 생성)
  - LangGraph 도입하여 홈페이지 헤더/푸터 일정하게 유지되도록 구현
  - SaaS 형태로 깔끔한 UI
  - 서빙 컨테이너 연동 예정
  - Test Mode 먼저 진행 예정
- 이미지 생성 시 시나리오별 구분: 사용자가 제품 사진을 넣었을 때와 넣지 않았을 때의 상황에 맞춰 이미지가 생성되도록 구현
- 사용자가 업로드한 사진의 분위기를 읽어내서, 그와 어울리는 배경 자동 추천 기능 시도 중
  - AI 스러운 느낌이 나면 광고효과가 떨어지지 않을지 의견 제시
  - 광고는 제품이 눈에 띄어야 하므로 배경과 제품이 어느 정도 분리되는 전단지 느낌 의견 제시
  - VM 성능 한계로, 일단 기능을 완성하고 프롬프트 조절로 해결 제안
- 테스트 중 서버가 끊기면 강제로 초기화할 수 있는 System Reset API 추가 (REST API 형태로 제공)
- 이미지 업로드 칸과 텍스트 입력 칸을 나눌지 합칠지 의견 요구
