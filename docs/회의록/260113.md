## 주요 안건 및 논의사항

### 1. 진행 현황 공유
**각자 진행 상황**:
- 김명환: MCP·서빙 서버 고도화 및 LLM 연동
- 김민혁: 이미지 내 텍스트 제거 및 재삽입 기능 구현
- 박지윤: 전체 흐름 파악, 진행 사항 및 일정 확인
- 이건희: 프론트엔드, 백엔드 구축
- 이솔형: 이미지 합성 품질 개선

### 2. 핵심 논의사항
**주요 이슈**:
- MCP·서빙 서버 고도화
  - 도커 및 서버 구성 보강
  - MCP -> AI 서빙 서버로 파라미터가 정확히 전달되는지 검증, 부족한 파라미터 전달 로직 수정 완료
  - 실시간 진행 상태(Progress) 추적 로직 점검
    - 단계별 퍼센트 값이 정상적으로 증가하는 것 확인
    - 각 단계가 순차적으로 실행되는 구조 확인
    - 최종 100%까지 도달하는 프로세스 정상 확인
- 테스트 흐름
  - 입력: 사용자 입력 프롬프트 (예: "바나나 광고 이미지 생성 요청 (이미지 + 텍스트)")
  - 처리: 광고 생성 작업 시작, MCP에 등록된 도구 순차적 실행, 각 단계별 진행률(progress bar) 갱신
  - 결과: 전체 파이프라인이 끝까지 정상 완료됨을 확인
- LLM 사용은 MCP 서버와의 규칙(프로토콜)을 반드시 따라야 함
  - 해당 규칙 문서화 진행 예정
- 실행 방식을 두 가지 모드로, 옵션 값(max calls 등)으로 제어
  - 즉시 실행 모드: 사용자 입력 1회, 추가 질문 없이 바로 전체 생성 파이프라인 실행
  - 대화 모드: LLM이 생성한 중간 프롬프트들(배경, 합성, 텍스트 효과 등)을 제시, "이대로 진행할까요?" 같은 확인 단계 가능
  - 현재 구현 상태에서는 중간 입력 없음. 입력 -> 처리 -> 출력 구조.
  - 챗봇 형태는 선택 사항. UX/UI 설계에 따라 한 번에 생성하는 방식/단계별 선택을 받는 방식 둘 다 가능하지만, 챗봇 UI는 구현 난이도가 높음.
- 이미지 내 텍스트 제거 및 재삽입 기능
  - OCR + 마스크 생성
  - Flux 모델 기반 Inpainting으로 텍스트 제거
  - 이후 텍스트 재삽입
  - 도커 기반 테스트 서버 진행 예정
- 점포 정보 입력 UI 구현
  - 모든 정보 입력 후, 홈페이지 생성하기 버튼 활성화 -> 버튼 클릭 시 홈페이지 생성 완료 메시지 표시 -> 생성된 홈페이지 URL로 즉시 이동 가능
  - AI 서빙 컨테이너 연동 예정
  - 현재는 이미지 입력만 가능
  - 이미지 + 텍스트 입력 시 이미지 생성 및 생성된 이미지를 홈페이지 갤러리에 추가 예정, SNS 홍보 버튼으로 바로 연동 예정
- 이미지 합성 품질 개선
  - 합성 강도(Blend strength) 최적화 진행 중
  - 외곽선 처리 및 그림자 품질 개선 작업 진행
